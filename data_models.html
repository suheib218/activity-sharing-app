Activity Manager Data Model

This document describes the data model for the Activity Manager app. The data is structured relationally using SQLAlchemy ORM models. Each entity is represented by a table, and relationships are defined between them.

Users

The Users table stores information about users who can create and join activities. 

{
  "id": "integer (primary key)",
  "username": "string (150 characters, unique, not nullable)",
  "email": "string (150 characters, unique, not nullable)",
  "password": "string (150 characters, not nullable)",
  "activities": "relationship to Activity (one-to-many)",
  "joined_activities": "relationship to Participation (one-to-many)"
}

Activities

The Activities table contains information about activities created by users. 

{
  "id": "integer (primary key)",
  "activity_name": "string (150 characters, not nullable)",
  "description": "string (300 characters, optional)",
  "date_time": "datetime (not nullable)",
  "organizer_id": "integer (foreign key to Users.id, not nullable)",
  "participants": "relationship to Participation (one-to-many)"
}

Participation

The Participation table records which users have joined which activities, ensuring that a user cannot join the same activity more than once. 

{
  "id": "integer (primary key)",
  "user_id": "integer (foreign key to Users.id, not nullable)",
  "activity_id": "integer (foreign key to Activities.id, not nullable)",
  "unique_participation": "constraint to ensure unique participation (user_id + activity_id)"
}

Relationships

  • User to Activity: A user can create multiple activities. This is a one-to-many relationship.
  • User to Participation: A user can join multiple activities, recorded in the Participation table.
  • Activity to Participation: An activity can have multiple participants. This is a one-to-many relationship.
