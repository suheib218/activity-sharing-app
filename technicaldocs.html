Overview

The Activity Sharing App is a Flask-based web application that allows users to create, join, and manage activities in real-time. The app is built using Flask for routing and backend logic, while SQLite is used as the database for storing user information, activity data, and participation details. The frontend is designed with Bootstrap for responsive layouts, using Jinja templates to render dynamic content.

Frontend

HTML Templates

The structure and content of web pages are defined using HTML templates. Each template corresponds to a specific part of the app, such as user registration, activity details, and the activity creation form. These templates are dynamically generated using Jinja2, allowing real-time data to be injected into the HTML content.

Bootstrap

Bootstrap is used to create responsive and visually consistent layouts for the application. It provides predefined CSS classes and JavaScript components for designing forms, buttons, and grids, enabling a clean and modern user interface without needing to write custom CSS from scratch.

Jinja Templates

Jinja2 is a templating engine integrated with Flask that allows dynamic rendering of HTML. It enables embedding Python logic directly into the templates, facilitating the display of user-specific data like activity details, profiles, and form fields. This dynamic generation of content makes the app interactive and user-specific.

Backend

Flask (Routing and HTTP Request Processing)

Flask handles the routing and HTTP request processing in the application. Each route is associated with a view function that processes incoming requests and returns a corresponding response. Routes are defined for various features such as creating an activity, joining an activity, user login, and viewing activity details.

Python (Business Logic)

The core functionality of the app is written in Python. It manages activity creation, updating, deletion, and user management. Role-based access control ensures that only the creator of an activity can modify or delete it. The business logic also includes server-side validation of forms to maintain data integrity and security.

SQLite (Database Interactions)

The app uses SQLite as its database for persisting user data, activity information, and user participation. SQLite is lightweight and easy to set up, making it ideal for this app. The database schema consists of tables for users, activities, and user-activity relationships. Queries are handled using SQLAlchemy to interact with the database in a secure and efficient manner.
